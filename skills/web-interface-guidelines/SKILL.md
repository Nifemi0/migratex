# Web Interface Guidelines

Use this skill whenever you design, build, or review a web interface. It centralizes the full checklist in `references/guidelines.md`.

## How to use
- For implementation: Read the relevant sections in `references/guidelines.md` before coding; keep hit targets, focus handling, and animation choices aligned with the checklist.
- For review: Skim all section headings, then deep-dive into any that apply (e.g., Forms, Animations, Performance). Flag violations and propose fixes.
- For regressions: Re-run the checklist on modified areas and verify keyboard/ARIA flows, loading states, and motion preferences.

## Bundled resources
- `references/guidelines.md` — Complete guideline text organized by topic.

## Output expectations
- Produce concise, actionable findings (what/where/why + fix) when auditing.
- When implementing, cite which checklist items you applied and note any intentional deviations.
